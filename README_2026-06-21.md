# EchoMind — Multimodal Emotion Intelligence

**Build log and project documentation**
Last updated: June 21, 2026

---

## What is EchoMind

EchoMind is a multimodal emotion AI system that reads emotion from three simultaneous input channels — text, webcam (facial expression), and audio — and fuses the outputs into a single unified emotional vector.

The core insight: the useful signal is not when all three modalities agree. It is when they conflict. A single-modality model never sees that disagreement. EchoMind does.

The system is currently in Phase 01. The frontend runs entirely in demo mode using mock model outputs. Real model integration (Phase 02) begins after the NLP fine-tune completes.

A single self-contained HTML file. Full browser application. No backend required. All emotion predictions come from JS mock functions.

Three-panel fixed-viewport layout (no page scroll):

**Zone A — Input (38% width)**
- Sub-panel 1: Text input with live character counter, keyword detection hints, Enter-to-analyze shortcut, weight slider

- Keyboard shortcuts row: ENTER / ESC / R / SPACE

**Zone B — Primary output (36% width)**
- Dominant emotion display: Bebas Neue at clamp(64px, 8vw, 100px), GSAP fade-swap animation on every update, colored by emotion token
- Animated confidence bar + percentage
- Chart.js radar chart with 400ms smooth transitions
- Live fusion weight bars synced to sliders in Zone A in real time
- FUSE + ANALYZE button
- Auto-normalization warning when weights do not sum to 100%
- Latency display after each run (TEXT Xms / FACE Xms / AUDIO Xms)

**Zone C — Secondary output (26% width)**
- Signal breakdown: seven horizontal confidence bars, all GSAP-animated simultaneously
- Emotion timeline: Chart.js line chart keeping last 20 entries across all 7 emotion channels
- Modality split: three independent columns showing top-3 predictions per model (TEXT / FACE / AUDIO) separately from the fused output
- Session log: newest-first, GSAP slide-in animation for new entries, timestamps

Header bar (52px):
- Live analyses count, current dominant emotion (colored), session MM:SS timer
- RESET SESSION clears all state and returns to neutral

Mock model layer (JS):
- `predictText(text)` — keyword-biased Dirichlet distribution, detects joy/sadness/anger/fear/surprise/disgust keywords
- `predictFace()` — sine-wave drift using `Date.now()` so the webcam feed looks live
- `predictAudio(hasInput)` — realistic random Dirichlet distribution
- `fusePredictions(text, face, audio, weights)` — weighted late fusion with auto-normalization
- All return the same shape: `{ emotions, dominant, confidence, latency_ms }` — identical to the inference function in `predict.py`

### 3. NLP training script — `train_nlp.py`

A standalone Python training script. No Jupyter. No Colab. Run with `python train_nlp.py`.

Target hardware: RTX 4050 Laptop, 6GB VRAM. Expected runtime: 60–90 minutes.

Model: `roberta-base` (125M parameters). Fine-tuned on Google's GoEmotions dataset (58,000 Reddit comments, 28 labels) remapped to 7 core emotion classes.

**7-class label mapping:**

| Class | GoEmotions labels mapped |
|---|---|
| joy | joy, amusement, excitement, admiration, approval, gratitude, love, optimism, pride, relief, caring |
| sadness | sadness, grief, disappointment, remorse, embarrassment |
| anger | anger, annoyance, disapproval |
| fear | fear, nervousness |
| surprise | surprise, confusion, curiosity, realization |
| disgust | disgust |
| neutral | neutral, desire |

**Training configuration:**

| Setting | Value | Reason |
|---|---|---|
| Model | roberta-base | 125M params, fits 6GB at batch 32 with fp16. roberta-large OOMs. |
| Batch size | 32 × 2 grad accum = 64 effective | Safe on 6GB |
| Learning rate | 1e-5 | More stable than 2e-5 for 5 epochs |
| LR schedule | cosine | Better convergence than linear for 5+ epochs |
| Epochs | 5 | Local GPU allows full run |
| FP16 | True | ~1.8x speedup, halves VRAM usage |
| Label smoothing | 0.1 | Prevents overconfidence, improves F1 on rare classes |
| Class weights | balanced | Corrects disgust (~2%) vs neutral (~35%) imbalance |
| dataloader_num_workers | 0 | Windows-safe (see below) |
| Early stopping | patience=2 | Stops if validation F1 plateaus for 2 epochs |

**Target metrics:** Weighted F1 >= 0.65, Accuracy >= 0.63

**Outputs saved to `./echomind-nlp-output/`:**
- `model/` — full fine-tuned model + tokenizer
- `training_curves.png` — 6-panel dark-themed chart (train loss, val loss, F1, accuracy, LR schedule, per-class F1)
- `confusion_matrix.png` — raw counts + row-normalized 7×7 heatmap
- `classification_report.txt` — per-class precision/recall/F1 to 4 decimal places
- `echomind_metadata.json` — full run config + test scores
- `predict.py` — standalone 15-line inference file, same return shape as JS mock

**`predict.py` return shape** (matches `predictText()` in `echomind-app.html`):
```python
{
  "emotions":   {"joy": 0.45, "neutral": 0.20, ...},  # all 7, sum to 1.0
  "dominant":   "joy",
  "confidence": 0.45,
  "latency_ms": 23.7
}
```

### 4. Windows multiprocessing fix

**Root cause:** Windows uses `spawn` (not `fork`) for child processes. When `dataloader_num_workers > 0`, PyTorch spawns worker processes that re-import the entire training script. Without a `if __name__ == "__main__"` guard, every worker re-executes `trainer.train()` before the main process finishes bootstrapping.

**Error:** `RuntimeError: An attempt has been made to start a new process before the current process has finished its bootstrapping phase.`

**Fix applied:**
```python
# All training logic moved into main()
def main():
    ...

if __name__ == "__main__":
    multiprocessing.freeze_support()  # required for frozen executables on Windows
    main()
```

Plus `dataloader_num_workers=0` in `TrainingArguments` — disables worker spawning entirely. On an RTX 4050 the GPU is the bottleneck, not the dataloader, so this has negligible training speed impact.

## How to run training

```bash
# 1. Create and activate virtual environment
python -m venv echomind-env
echomind-env\Scripts\activate

# 2. Install PyTorch with CUDA 12.1 (RTX 4050)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

# 3. Install dependencies
pip install transformers==4.44.0 datasets==2.20.0 evaluate==0.4.2
pip install accelerate==0.33.0 scikit-learn==1.5.1
pip install matplotlib==3.9.0 seaborn==0.13.2 huggingface_hub==0.24.5

# 4. Set HuggingFace token (for Hub push at end of training)
set HF_TOKEN=your_token_here

# 5. Run
python train_nlp.py
```

Expected output: `./echomind-nlp-output/` with model, charts, report, and `predict.py`.
