WHAT I
AM BUILDING

Most emotion AI on GitHub reads one signal. Text sentiment. Or a face. Or audio. I wanted to know what the disagreement between them looks like.

EchoMind takes all three simultaneously — runs them through separate models, then fuses the outputs into a unified emotional vector. The interesting output is not consensus. It is conflict.

INPUT SIGNALS Three streams. One output.
SIGNAL 01 / TEXT
TEXT

Reads typed or transcribed text. Fine-tuned RoBERTa model on the GoEmotions dataset. 27 labels collapsed to 7 core emotional states.

MODEL RoBERTa-base DATASET GoEmotions LATENCY ~120ms OUTPUT 7-dim vector
SIGNAL 02 / VOICE
VOICE

Processes raw audio waveforms. Wav2Vec2 feature extraction combined with MFCC and chroma features. Trained on the RAVDESS emotional speech dataset.

MODEL Wav2Vec2 FEATURES MFCC · Chroma LATENCY ~90ms OUTPUT 7-dim vector
SIGNAL 03 / FACE
FACE

Analyzes live webcam frames. ResNet-50 fine-tuned on FER2013 and AffectNet. OpenCV handles frame capture and preprocessing before inference.

MODEL ResNet-50 INPUT OpenCV frame LATENCY ~35ms OUTPUT 7-dim vector

THE DISCOVERY

The useful signal
is not when all three agree.

When your words say neutral but your face says anger — that conflict is information. A single-modality model never sees it.

The fusion layer does not pick a winner. It weighs the disagreement and that is where EchoMind is different from every other emotion project on GitHub.

TEXT calm
VOICE calm
FACE tense
CONFLICT DETECTED
dominant: anger / 0.67
Weighted fusion: text(0.4) + voice(0.35) + face(0.25)
